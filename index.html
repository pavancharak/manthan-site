The Proof
Every decision leaves
a signed record.
01
Matched rule is the only source of truth

The explanation a voice agent reads back comes exclusively from the rule that matched. No AI paraphrasing, no post-hoc generation, no embellishment.

02
Proof hash + signature tamper-evident

The token is signed with a cryptographic proof_hash. Any modification — of the outcome, explanation, or inputs — breaks the signature.

03
Replay any past decision exactly

Original schema + rule snapshot are versioned and immutable. Any past decision can be re-run and will produce the identical outcome — provably.

04
event_id prevents duplicate execution

The execution gateway verifies the token and rejects replays via event_id. An ALLOW decision cannot be used twice.

Core Guarantees
What Manthan promises.
Not aspirationally. Architecturally.
→
Determinism

Same Decision Input, same Decision Output. No LLM variance, no context drift, no mood. The engine is pure logic.

⊘
No AI in evaluation

AI structures input and reads output. It never enters the evaluation path. That boundary is architectural, not policy.

↯
Fail-closed

Error, uncertainty, incomplete data, schema mismatch the answer is always BLOCK. ALLOW is never the default.

⬡
Signed tokens

Every decision is a cryptographically signed token. Execution is gated on valid signature + event_id. Replay is prevented.

⟳
Reproducibility

Schema + rule snapshots are immutable. Any past decision can be re-run identically. History is auditable forever.

≡
Versioned evolution

No hidden defaults. No silent changes. Every schema or rule change requires a version bump. No retroactive mutation.
