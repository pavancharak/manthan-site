Manthan

Overview Pipeline Architecture Schema Rules Token Constraints

Overview

Manthan is a deterministic decision infrastructure. Identical input produces identical output. Always.

Pipeline

Evaluation is executed in strict order:

1. Validation     → INVALID
2. Completeness   → INCOMPLETE
3. Evaluation     → ALLOW / BLOCK / ESCALATE
4. Explanation    → TRACE

Architecture

Definition → Schema + Rules
Signal     → External deterministic inputs
Core       → Pure evaluation
Token      → Signed proof
Gateway    → Execution control

Schema

Defines valid input structure. Strict and versioned.

{
  "type": "object",
  "required": ["amount", "role"],
  "properties": {
    "amount": { "type": "number" },
    "role": { "enum": ["admin", "user"] }
  }
}

Rules

Deterministic evaluation logic.

{
  "rule_id": "allow-standard",
  "if": [
    { "field": "amount", "op": "<=", "value": 1000 },
    { "field": "role", "op": "==", "value": "admin" }
  ],
  "then": "ALLOW"
}

Decision Token

{
  "event_id": "evt_...",
  "outcome": "ALLOW",
  "rule_version": "v1",
  "schema_version": "v1",
  "matched_rule": "allow-standard",
  "proof_hash": "sha256:...",
  "signature": "ed25519:..."
}

Hard Constraints

- No hidden defaults
- No runtime mutation
- No AI in evaluation
- No bypass of pipeline
- No retroactive change
